# MeetMate 项目开发计划说明

## 项目目标

开发一个简洁、可运行、可演示的多人智能排期系统，覆盖用户登录、个人日程、Group 管理、活动排期、智能推荐和活动同步等核心流程。

## 开发阶段

### 第一阶段：需求分析

产出：

- 明确用户角色：普通用户、Group 创建者。
- 明确核心业务流程：注册登录、个人日程、Group 管理、排期活动、推荐确认。
- 明确 MVP 边界：不做真实短信、邮件、第三方日历和实时协作。

### 第二阶段：系统设计

产出：

- 技术栈：Python + Streamlit + SQLite。
- 模块划分：`app.py`、`database.py`、`auth.py`、`models.py`、`scheduler.py`、`ui_helpers.py`。
- 数据库表设计：用户、Group、成员关系、个人日程、Group 活动、空闲时间。

### 第三阶段：编码实现

产出：

- 登录注册与密码哈希。
- SQLite 自动初始化。
- 个人日程 CRUD。
- Group 创建和邀请码加入。
- 待排期活动创建。
- 成员空闲时间提交。
- 智能排期评分和推荐理由。
- 活动确认与同步。

### 第四阶段：测试验证

产出：

- 注册、登录、重复注册测试。
- Group 创建、加入、重复加入测试。
- 日程添加、删除、冲突检测测试。
- 活动创建、空闲时间提交、智能推荐测试。
- 活动确认和个人日程同步测试。

### 第五阶段：课堂展示

展示重点：

- 多人找时间困难这一真实问题。
- 系统如何统一收集成员空闲时间。
- 智能排期算法如何计算得分。
- 个人日程冲突如何影响推荐结果。
- 确认后的 Group 活动如何同步到成员个人日程。

## 进度安排

| 阶段 | 时间 | 任务 |
| --- | --- | --- |
| 需求分析 | 第 1 天 | 阅读文档，整理功能边界 |
| 系统设计 | 第 1 天 | 完成数据库和模块设计 |
| 核心开发 | 第 2-3 天 | 实现后端逻辑与 Streamlit 页面 |
| 测试修复 | 第 4 天 | 完成主要测试用例并修复问题 |
| 文档整理 | 第 5 天 | 完成 README、测试用例和展示材料 |

## 风险与应对

| 风险 | 影响 | 应对 |
| --- | --- | --- |
| 页面流程过多 | 演示时容易迷路 | 使用左侧导航和清晰按钮入口 |
| 排期算法难以解释 | 展示效果变弱 | 在推荐表中展示得分、人数和理由 |
| 数据重复同步 | 个人日程出现重复活动 | 使用 `source_activity_id` 检查已有同步记录 |
| 时间冲突判断出错 | 推荐结果不可信 | 使用标准重叠判断：`start_a < end_b 且 start_b < end_a` |
