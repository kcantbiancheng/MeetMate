# MeetMate 测试用例

## 1. 注册测试

输入：

```txt
用户名：test01
密码：123456
昵称：测试用户
```

预期结果：

```txt
注册成功，users 表新增记录，密码以哈希形式保存。
```

## 2. 重复注册测试

输入：

```txt
用户名：test01
密码：abcdef
昵称：重复用户
```

预期结果：

```txt
系统提示该用户名已被注册，不新增用户。
```

## 3. 登录测试

输入：

```txt
用户名：test01
密码：123456
```

预期结果：

```txt
登录成功，进入首页，session state 保存登录用户。
```

## 4. 登录失败测试

输入：

```txt
用户名：test01
密码：wrong
```

预期结果：

```txt
系统提示用户名或密码错误。
```

## 5. 创建 Group 测试

输入：

```txt
Group 名称：工程概论项目组
Group 类型：项目组
Group 描述：用于课程项目排期
```

预期结果：

```txt
创建成功，系统生成 6 位邀请码，创建者自动成为 Group 成员且角色为 owner。
```

## 6. 加入 Group 测试

输入：

```txt
邀请码：创建 Group 时生成的邀请码
```

预期结果：

```txt
用户成功加入对应 Group，并能在“我的 Group”中看到该 Group。
```

## 7. 重复加入 Group 测试

输入：

```txt
同一用户再次输入同一个邀请码
```

预期结果：

```txt
系统提示已经加入该 Group。
```

## 8. 添加个人日程测试

输入：

```txt
标题：Java 作业
日期：2026-06-12
开始时间：19:30
结束时间：20:30
地点：图书馆
类型：个人
```

预期结果：

```txt
日程添加成功，并在个人日程列表中按时间排序显示。
```

## 9. 日程时间校验测试

输入：

```txt
标题：错误日程
开始时间：20:30
结束时间：19:30
```

预期结果：

```txt
系统提示日程开始时间必须早于结束时间。
```

## 10. 创建待排期活动测试

输入：

```txt
活动名称：项目讨论会
候选日期：2026-06-12 至 2026-06-13
候选时间：19:00 至 22:00
活动时长：60 分钟
地点：线上会议
```

预期结果：

```txt
活动创建成功，系统可以生成 6 个候选时间段。
```

## 11. 成员提交空闲时间测试

输入：

```txt
用户 A：2026-06-12 19:00 有空，20:00 没空
用户 B：2026-06-12 19:00 有空，20:00 有空
用户 C：2026-06-12 19:00 不确定，20:00 有空
```

预期结果：

```txt
availability 表保存每个成员对每个候选时间段的选择。
```

## 12. 智能排期测试

输入：

```txt
多个成员的空闲时间数据
```

预期结果：

```txt
系统按综合得分从高到低显示候选时间段，并展示有空人数、不确定人数、没空人数、冲突人数和推荐理由。
```

## 13. 个人日程冲突测试

已有个人日程：

```txt
2026-06-12 19:30-20:30 Java 作业
```

候选时间：

```txt
2026-06-12 19:00-20:00
```

预期结果：

```txt
系统识别两个时间段重叠，对该候选时间段扣除冲突惩罚。
```

## 14. 确认活动测试

操作：

```txt
Group 创建者选择推荐时间并点击确认活动。
```

预期结果：

```txt
活动状态变为 confirmed，final_date、final_start_time、final_end_time 被写入，并同步到所有 Group 成员的个人日程。
```

## 15. 重复确认同步测试

操作：

```txt
已确认活动再次触发同步逻辑。
```

预期结果：

```txt
系统更新已有同步日程，不重复插入相同 source_activity_id 的个人日程。
```
